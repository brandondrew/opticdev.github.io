Writing Skills

You can teach Optic to do just about anything with your source code. Teams have built support for many popular libraries, to generate their standard tests and one group even migrated to GraphQL using Optic.

Before we get much deeper it's important to familiarize yourself with the first class citizens in Optic's Skill API and how they all fit together. Aidan made a 7 minute video introducing these concepts that we suggest you watch before proceeding.

Skills are written in Optic Markdown and we offer tooling that makes building and testing Optic Markdown much easier.

Optic Markdown

Most of the world’s programming knowledge is kept in unstructured formats that are great for humans and useless to machines. This is unfavorable because there are many practical development tasks that could be automated if computers only knew what to do. Instead of trying to make our own standard and require people to maintain a whole other set of files, we decided to meet programming knowledge where it lives today: markdown.

Optic Markdown relies on simple, invisible annotations in markdown to define Schemas, Lenses, and Transformations. Optic Markdown has been designed to store programming knowledge in a way that makes it accessible to be both humans and smart tools. You can create new documentation using Optic Markdown or you can update existing markdown documentation to work with Optic.

Compilation

The Optic Markdown you include in your project goes through several stages before it is ready to be used by Optic.

  1. Parse Phase: Annotations and sample code are extracted from the markdown file
  2. Resolve Phase: Optic Markdown can reference objects in other Optic Markdown files. In this phase all such dependencies are resolved.
  3. Compile Phase: Schemas & Transformations are validated. Optic generates a Parse & Render method for each Lens.
  4. Knowledge Graph Creation: A knowledge graph (like the one in the video) with edges between every Schema, Lens and Transformation is saved. This graph is used in most of the queries the API runs.

Tooling

The optic-markdown npm package contains a GUI and other tooling that makes teaching/debugging Optic easier.

You can validate an Optic Markdown file has a valid format by running:

opticmd /path/to/file

If you want to use a GUI that makes it easier to build Lenses and Transformations you can use the trainer

opticmd starttrainer

Fast Feedback

If you want to develop Optic docs with live feedback, add them to an Optic project by listing their parent directory in your optic.yml file and importing your package with its identifier. Optic’s package manager resolves packages locally first before it looks in public/private Optic registries, so all of the changes you make here apply within a few seconds.

knowledge_paths:
  - /docs

skills:
  - username:new-package

You can test if your skills packages work by using Optic normally or by opening up our experimental debugger (Tools -> Debugger in the main menu).

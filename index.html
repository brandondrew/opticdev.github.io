What it Does

1. Write live docs for your project

Document your best practices, patterns and other knowledge in markdown.

2. Import docs shared by other users

This includes knowledge from your teammates, a best practice shared by a developer you look up to, or docs for a library you want to use.

3. Ask Optic for Help

Optic uses the imported knowledge to generate/modify code.

4. Optic Answers with Real Code

After approval, Optic pastes the solutions directly into your code

Project Goals

  1. Organize the world’s programming knowledge and make it accessible through automation
  2. Elevate developer focus to their most important work
  3. Empower teams to share knowledge and enforce conventions
